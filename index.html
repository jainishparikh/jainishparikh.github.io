Roaring Bitmap Compression Engine 6sense · 2026

The problem: a high-cardinality dataset at 10 billion rows was making query workloads intractable. I designed a multi-dimensional compression architecture using Roaring Bitmaps that collapsed it to 200,000 rows — a 50,000× reduction — without sacrificing queryability.

The harder problem: SingleStore, Hive, and Trino don't natively support bitmap AND/OR operations. I wrote native Rust database extensions — UDFs, UDAFs, and TVFs — that push bitwise computation directly into the query engine, eliminating intermediate row materialization entirely. The result was a database-agnostic compression architecture that cut pipeline processing time by 40% (5 hrs → 3 hrs) and delivered measurable infrastructure cost reduction.

JSON-to-SQL Query Compiler 6sense · 2024–2026

I designed and owned 6sense's query compilation layer from scratch: a source-to-source transpiler that parses declarative JSON schemas into an abstract syntax tree, runs multi-pass optimization passes — compression, deduplication, predicate restructuring — and emits production-ready SQL powering multiple customer-facing products across terabyte-scale datasets.

What started as an internal tool became the platform primitive that multiple engineering teams build on. On top of it, I built a Django service with a React/TypeScript frontend that surfaces query capabilities to internal product teams end-to-end. Optimizing the pipelines beneath it — 200K+ nightly queries — yielded 1.5× faster average runtimes and 5× latency reduction on heavy-tail queries.
